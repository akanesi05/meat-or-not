# 🥩 渡辺さんの嫌いな肉クイズ（meat-or-not）

Djangoで作る食材判定クイズアプリ  
**「渡辺さんにとって、その肉は食べられるか？」** をテーマにしたクイズです。  
シンプルな操作で、ちょっと笑えて学べる（？）語彙力アップアプリ。

---

## 🧠 概要

職場で大変お世話になっている渡辺さんは、肉が苦手です。  
特に「鳥」が大の苦手で、帰宅時に飛んでいる鳥を見ても怯え、  
モニターに鳥が映ると今日も叫んでいました。
でも、実は「食べられる肉」もあるそうです。  
肉好きの私としては思想が真逆で、そのこだわりが面白いと思い、  
**クイズアプリとして実装** してみました。

---

## 🎮 Features

- 出題された食べ物が「渡辺さんにとって肉かどうか」を ⭕️❌ で答えるクイズ形式  
- 渡辺さんが嫌いな肉（特に「鳥」）が含まれているかを自動判定  
- Django を使用したバックエンド構成  
- （将来的に React でフロントを実装予定）

## 🚀 開発ロードマップ（meat-or-not）

### 🥇 MVP（最小実装版）
まずは「動くクイズ」として完成させる最小構成です。

| 機能 | 内容 | 状態 |
|------|------|------|
| 出題 | データベースの Food からランダムに1件出題 | 🔴 完了 |
| 正誤判定 | ⭕️❌ボタンで回答 → 判定結果を返す |  🔴 完了 |
| 結果表示 | 正解／不正解メッセージを表示 | 🟢 設計中 |
| 結果共有 | SNS（Xなど）に結果をシェアできる機能 | 🟡 構想中 |

---

### 🥈 次フェーズ：UI・UX拡張（React化）
ユーザー体験を向上させるため、DjangoバックエンドをAPI化してReactで画面構築を行います。

| 機能 | 内容 | 状態 |
|------|------|------|
| フロントエンドReact化 | DjangoのAPIをReactで呼び出す構成に変更 | 🟡 構想中 |
| SPA構成 | ページ遷移をなくし、スムーズなクイズ体験 | 🟡 構想中 |
| API化 | Django REST Frameworkを導入し、JSONでやりとり | 🟡 構想中 |

---

### 🥉 将来的な拡張（ランキング・クイズ一覧）
MVPの完成後、継続的なプレイを促すための拡張機能を追加します。

| 機能 | 内容 | 状態 |
|------|------|------|
| ランキング機能 | 正答数・連続正解数などでランキング表示 | ⚪️ 企画中 |
| クイズ一覧ページ | 過去の出題（食材一覧）を閲覧可能にする | ⚪️ 企画中 |
| 管理画面強化 | 食材の追加・編集・カテゴリ管理を充実化 | 🟢 設計中 |

## 🚀 インストール方法

```bash
# リポジトリをクローン
git clone https://github.com/akanesi05/meat-or-not.git
cd meat-or-not

# 依存関係をインストール
pip install -r requirements.txt

# マイグレーション実行
python manage.py migrate

# ローカルサーバー起動
python manage.py runserver
